为什么要学习Git？

Created At：2023-12-21 07:17 Updated At：2023-12-21 16:01
Keywords: blog git

因为Git是最常用的版本管理工具，写博客要用。
因为不清楚、不熟悉Git的原理和使用，我们难以持续地对博客进行更新。

因为博客框架主题等不断在更新，这就要求熟悉Git，另外我们需要创建分支对博客源文件进行备份。如果没有备份，本地源文件丢失，例如换电脑或者重装系统等或者无意中被损坏了，那么博客就得从头再来，以前写的就付诸东流了，没有安全感。就难以持续地投入写作、更新博客。

为什么要写博客？

记笔记是为了记录个人的想法、一些需要记住的，方便查看的东西，不一定成体系。

而博客是输出内容，是有潜在的读者的，因此就可以整理思路，给别人讲清楚。这样对自己绝对是一个 锻炼。
在分享的同时，也整理了自己的思维。
把零散的知识系统化。

一个东西自己模糊地可能觉得明白了，但是只有在尝试给别人讲明白时，才是真正检测自己是否真的清楚明白的时候。

比如我现在写的这篇笔记，也可以说是博客的预备内容，就是在整理思路。

虽然一时还不完美（思维具有跳跃性，自己知道自己在说什么，但是别人不一定明白。因此需要写给别人看就需要交代出来。把事情说清楚，并且不要太啰嗦，冗长，让人抓不住重点。）

通过这样的练习，就能够发现自己思维和表达时的不足，然后才能提高。

那么，对于写博客来说，由于我们使用Hugo+Github Pages作为博客的框架和平台。为什么？因为Hugo是目前最流行的静态博客框架，而且Github Pages可以免费托管。
为什么不使用CSDN, cnblogs等国内平台来写？
你懂的，不想多说。

那么使用Hugo和Github Pages写博客需要的工具集合里Git就是关键的一个。
其作用是（引用自官网：

Git is required to:
Build Hugo from source
Use the Hugo Modules feature
Install a theme as a Git submodule
Access commit information from a local Git repository
Host your site with services such as CloudCannon, Cloudflare Pages, GitHub Pages, GitLab Pages, and Netlify

需要用Git的地方：
从源码构建Hugo
使用Hugo模块特性
安装一个主题作为一个Git submodule
从本地Git仓库中访问提交信息
通过服务例如CloudCannon, Cloudflare Pages, GitHub Pages, Gitlab Pages以及Netlify提供主机服务。

但是Git的功能特性较为复杂，时间精力有限，我们只学习博客搭建和使用过程中会用到的功能特性以及部分原理。

即围绕着Hugo博客的搭建来学习git，侧重于使用，如果对于使用有帮助，也将学习必要的原理。
