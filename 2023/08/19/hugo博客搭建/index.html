Hugo博客搭建

By Adosa

August 19, 2023

写博客

写博客和记日记、写总结很重要 能够帮助我们理清思路 博客也能够帮助到其他人. 看了这位作者的文章《记笔记与写博客》，更激发了自己写博客的想法:

博客和笔记有什么区别呢？一些技术性内容，笔记是散碎的，不一定是正确的，只要自己能看懂就可以，而写出来的文章是要被别人看到的，因此条理清晰与内容准确就是一个体面博主对自己行文的最基本要求。所以从这个角度看，记笔记很简单，但要把笔记整理成一篇融合自己观点的文章就不那么容易了，至少自己要完全理解才能写得出来…

我觉得内敛型的人都应该拥有自己的博客，或其它形式的分享平台，这样即使不喜欢日常与陌生人面对面交流内心深处的情感，至少还有一个地方可以把它们写出来，就像我的这个博客，我知道有人能看到，不论是我的朋友还是陌生人。

确实有这种感觉，作者说的很到位，深深赞同。自己也会写笔记，但是只是给自己看，所以就不会系统得写清楚来龙去脉，只要自己能看懂就行。

为什么不采用主流的商业博客平台呢？ 因为商业平台不可控,例如csdn常常需要登录才能复制代码，而我想拥有自己可以定制的一个博客，不受平台限制，可以发布例如佛法内容、可以下载文件等等。

自己搭建，也有选择，就是一种是自己租服务器，另外就是使用类似Github Pages这样的免费博客托管平台。 由于现阶段比较穷，服务器需要长期续费，不续费就gg了，所以暂不考虑。

博客主要是静态页面，不需要太多交互，那么流行的支持Github Pages的静态博客框架如下： hexo, hugo

之前试过hexo了，模板很多，但是如果长时间没升级，就容易挂掉。而且hexo博客相对来说生成速度比较慢。

hugo速度比较快，修改实时预览 因此博客将会采用hugo，另外一个原因是尊者之前要我帮忙弄国际巴利三藏学院的主页，也是hugo的。 熟悉了hugo之后可以更方便得服务。

为了更全面得使用hugo，因为需要将hugo博客部署到github page上，github使用git作为工具，所以需要先学习（复习）git的原理和基础使用。 之后再进行博客的部署 使用的是廖雪峰的git教程，以前学过，但是长时间不用，已经忘得差不多了。

git基本用法

git config --global user.email "you@example.com"
git config --global user.name "Your Name"

常用git命令

git init 初始化一个git仓库 git log 查看提交历史（HEAD指向的就是当前版本） git log --pretty=oneline 查看提交历史，美化显示 git commit 提交（暂存） git reset --hard HEAD^ 回退到上一个版本 git reset --hard COMMITID 回退到指定版本（COMMITID替换成指定的版本号） git reflog` 查看历史commit记录, 方便后悔切换回最新版本

暂存区是Git⾮非常重要的概念，弄明⽩白了暂存区，就弄明⽩白了Git的很多操作到底干了什么。 暂存区存储git add的文件，commit会将暂存区所有文件一次性提交到当前分支上。 ⽤用git diff HEAD -- readme.txt命令可以查看⼯

因为创建、合并和删除分⽀支⾮非常快，所以Git⿎鼓励你使⽤用分⽀支完成某个任务，合并后再删掉分⽀支，这和直接在master分⽀支上⼯工作效果是⼀一样的，但过程更安全。 ⼩小结 Git⿎鼓励⼤大量使⽤用分⽀支： 查看分⽀支：git branch 创建分⽀支：git branch name 切换分⽀支：git checkout name 创建+切换分⽀支： git checkout -b name 合并某分⽀支到当前分⽀支： git merge name 删除分⽀支： git branch -d name 暂存工作区：git stash 查看已暂存的工作区: git stash list

⼯工作现场还在，Git把stash内容存在某个地⽅方了，但是需要恢复⼀一下，有两个办法： ⼀一是⽤用git stash apply恢复，但是恢复后，stash内容并不删除，你需要⽤用git stash drop来删 除； 另⼀一种⽅方式是⽤用git stash pop，恢复的同时把stash内容也删了： 查看远程仓库详细信息：git remote -v 推送分支到远程仓库： git push origin dev(推送dev分支到远程仓库)

你的⼩小伙伴要在dev分⽀支上开发，就必须创建远程origin的dev分⽀支到本地，于是他 ⽤用这个命令创建本地dev分⽀支：

git checkout -b dev origin/dev

现在，他就可以在dev上继续修改，然后，时不时地把dev分⽀支push到远程

在本地创建和远程分⽀支对应的分⽀支，使⽤用git checkout -b branch-name origin/branchname，本地和远程分⽀支的名称最好⼀一致； • 建⽴立本地分⽀支和远程分⽀支的关联，使⽤用git branch --set-upstream branch-name origin/branch-name；

hugo 安装

Ubuntu安装Hugo

sudo apt update
sudo apt install hugo

查看hugo版本：

hugo version

快速创建一个博客站点：

hugo new site quickstart
cd quickstart
git init
git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke
echo "theme = 'ananke'" >> hugo.toml
hugo server

修改配置文件,

cd quickstart
vim config.toml
